import SwiftUI
struct ContentView: View {
@State var n1=""
@State var n2=""
@State var result=0
var body: some View {
        VStack {
            Text("Simple Calculator")
            TextField("Enter first number", text:$n1)
            TextField("Enter first number", text:$n2)  
        HStack (spacing:15) {  
                Button("+") {
        if let a=Int(n1), let b=Int(n2) {
        result=a+b
                    }
                }
                Button("-") {    
        if let a=Int(n1), let b=Int(n2) {
        result=a-b
                    }
                }
                Button("*") {
        if let a=Int(n1), let b=Int(n2) { 
        result=a*b
                        }
                       }
                Button("/") {     
        if let a=Int(n1), let b=Int(n2) {
        result=a/b
                    }
                }     
            }
        .buttonStyle(.borderedProminent)
        Text("Result: \(result)")
    }
  }
}

SwiftUI Tic-Tac-Toe — Docs

Purpose: Simple two-player Tic-Tac-Toe to practice SwiftUI state, view composition, and basic game logic.

Inputs/Outputs: Input = tile taps; Output = board updates, status text (turn, win, draw), and a restart action.

Design: Each tile stores its state message1…message9, current player toggles between X O, and checkWinner() evaluates winner. resetBoard() clears the state.

Debugging & Testing: Tested row/column/diagonal wins, full-board draw, and invalid repeat taps after game over.

Binary Counter

In this project, I learned how to represent numbers in both decimal and binary using SwiftUI’s state variables and conditional rendering. Creating the colored boxes for each binary bit helped me understand how values map to powers of two. Implementing the Enroll and Reset buttons taught me how to manage user input and update the interface dynamically. The overflow feature was especially fun — it made me think about how real systems handle limited data sizes.
